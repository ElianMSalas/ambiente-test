Chips

Enlace al componente original

Descripción

Los chips son elementos visuales compactos, parecidos a etiquetas o botones pequeños, que comunican una entrada, atributo o acción, representando información breve de forma clara y fácil de identificar.

Uso del componente

Los chips permiten mostrar etiquetas, categorías o estados, así como realizar acciones simples, como seleccionar, filtrar o eliminar información.

Diseño y Formato del componente

Los chips se presentan como formas rectangulares redondeadas y se componen de dos elementos clave: un contenedor y un texto de etiqueta, también pueden incluir íconos.

Es importante que el texto de la etiqueta sea claro y conciso, y que comparta el mismo estilo tipográfico que los botones.

Criterios de accesibilidad

Criterios Transversales

  1. Contraste: El texto dentro del chip debe ser legible y con contraste adecuado.

  2. Navegación por teclado: Los chips deben ser navegables con Tab y eliminables con Delete o Backspace.

  3. Atributos ARIA: Usar aria-label para chips sin texto visible.

  4. Compatibilidad con lectores de pantalla: Los chips deben ser interpretables como elementos seleccionables. Para chips eliminables, el botón de cierre debe tener aria-label="Eliminar [nombre del chip]".

Criterios Específicos

  1. Etiquetado HTML Semántico: Utilizar listas cuando los chips representen opciones seleccionables.

  2. Etiqueta visible: Asegurar que el texto de la etiqueta sea descriptivo, para que las personas usuarias puedan comprender el propósito sin un contexto adicional.

  3. Foco visible: Si hay un chip interactivo (por ejemplo: que permite eliminar o seleccionar) sean accesibles mediante el teclado y tengan foco visible.

  4. Gestión de estados: Si el chip tiene estados (como seleccionado, desactivado, etc.), estos deben ser perceptibles de manera visual y también comunicarse a los lectores de pantalla mediante atributos ARIA, como aria-selected o aria-disabled.
