Buttons (Botones)

Enlace al componente original

Descripción

Los Botones son unos de los elementos básicos de la interfaz de usuario que permiten la interacción con la aplicación. Este componente ejecuta una acción cuando es presionado, redirige a otras páginas, o dispara eventos en la aplicación.

Uso del componente

Los botones deben ser utilizados para realizar acciones como enviar un formulario, confirmar una selección, iniciar un proceso o navegar a otra sección. Es importante priorizar su uso para evitar sobrecargar la interfaz con demasiados botones y garantizar una jerarquía visual clara.

Diseño y Formato del componente

Los botones se presentan como formas rectangulares redondeadas y que contienen una etiqueta o llamado a la acción, donde también pueden incluir iconos.

Las etiquetas de los botones expresan qué acción se producirá cuando se interactúe con ellos. El texto debe ser claro y describir la acción que el usuario realizará.

  • Se deben utilizar verbos en infinitivo, como por ejemplo: "Ingresar", "Ir a pagar", "Solicitar Beneficio".
  • Si el texto son tres o menos palabras se utiliza el formato Titular; con mayúsculas en las iniciales de las palabras, excepto artículos y preposiciones, por ejemplo: Ir a Pagar.
  • Si el texto son cuatro o más palabras se utiliza el formato Sentencia; con mayúscula sólo en la primera letra, a excepción de nombres de productos, por ejemplo: Solicitar certificado de deuda de Contribuciones.

Si utilizarás solo un icono sin etiqueta, siempre debe ir con un aria-label. Además, si no puedes pensar en un icono apropiado para la acción a representar dentro de 5 segundos, es mejor que sólo se utilice texto para el botón (Regla de los 5 segundos). "Ante cualquier duda, el mejor icono es un texto"

Criterios de accesibilidad

Criterios Transversales

  1. Contraste: Los botones deben cumplir con los requisitos de contraste según WCAG 2.1 AA.

  2. Navegación por teclado: Los botones deben ser navegables con la tecla TAB y activables con las teclas Enter y Space.

  3. Atributos ARIA: Los botones deben ser accesibles por defecto sin necesidad de agregar atributos ARIA adicionales. Sin embargo, en algunos casos específicos, es recomendable incluirlos, ejemplo:

    • Si el botón no tiene un texto visible, debe agregarse aria-label o aria-labelledby para describir su propósito.
    • Si el botón activa un contenido dinámico (como mostrar u ocultar un menú), se puede usar aria-expanded y aria-controls para indicar el estado.

  4. Compatibilidad con lectores de pantalla: Los botones deben tener etiquetas claras y descriptivas, ejemplo: button aria-label="Guardar cambios"

Criterios Específicos

  1. Gestión de estados: Cuando un botón está deshabilitado, debe ser marcado con el atributo HTML "disabled", además de tomar la apariencia correspondiente.

  2. Foco visible: El estado de foco (cuando el botón recibe la interacción de un teclado o dispositivo de asistencia) debe ser claro. Se recomienda un borde visible o un sombreado suave alrededor del botón.

  3. Etiquetado HTML Semántico: Es importante utilizar correctamente las etiquetas de a o button al momento de generar un botón:

    • Se debe utilizar la etiqueta button para cualquier interacción que realice una acción en la página actual.
    • Se debe utilizar la etiqueta a para cualquier interacción que navegue hacia otra página o dirección o también para descargar algún recurso, ejemplo: un menú de navegación o descargar algun PDF o CSV. También debe siempre tener el atributo "href".
