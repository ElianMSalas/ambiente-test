Descripción

El Badge es un componente visual que se utiliza para resaltar información breve de manera adicional, como notificaciones, estados o contadores, de manera compacta y destacada. Se muestra generalmente como un pequeño indicador sobre otro componente, como botones, iconos o avatares para resaltar cambios o eventos importantes.

Desde el punto de vista de accesibilidad, es importante que los Badges sean anunciados correctamente por lectores de pantalla y que tengan suficiente contraste para ser perceptibles por todas las personas usuarias y que proporcione información textual alternativa cuando sea necesario.

Uso del componente

El Badge se usa en situaciones donde se requiere mostrar información adicional sin sobrecargar la interfaz. Es útil en:

  • Notificaciones o alertas sobre elementos interactivos.
  • Indicadores de cantidad en carritos de compra o listas de mensajes.
  • Como indicador visual en etiquetas de estado (ejemplo: "Nuevo", "En revisión").
  • Asociado a iconos o botones para dar contexto sin necesidad de texto adicional.

Buenas prácticas

  • No abusar del uso de badges para evitar sobrecargar la interfaz.
  • Asegurar que el badge no sea el único medio para transmitir información (se debe complementar con un texto visible o atributo ARIA cuando sea necesario).
  • Utilizar colores y tamaños adecuados para mantener la jerarquía visual clara.

Diseño y Formato del componente

El Badge se muestra como una insignia superpuesta a otro elemento (como un botón o icono).

Criterios de accesibilidad

Criterios Transversales

  1. Contraste: El texto dentro del badge debe cumplir con un contraste mínimo de 4.5:1.

  2. Navegación por teclado: Si el badge es interactivo, debe ser navegable con Tab y activable con Enter o Space.

  3. Atributos ARIA:

    • Usar aria-label o aria-labelledby para proporcionar información adicional cuando el badge transmite un dato importante.
    • Si el badge solo es decorativo y no aporta información adicional, marcarlo con aria-hidden="true".

  4. Compatibilidad con lectores de pantalla

    • Asegurar que la información representada por el badge pueda ser anunciada por tecnologías de asistencia.
    • Para badges con conteo, describir su propósito con aria-live="polite" si es dinámico.

Criterios Específicos

  1. Foco visible: Si el badge es interactivo, debe contar con un estado de foco claro y distinguible.

  2. Asociación con el elemento principal: Si el badge acompaña a un botón o ícono, debe estar correctamente relacionado semánticamente con aria-labelledby o aria-describedby.
