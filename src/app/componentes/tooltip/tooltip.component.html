Tooltip

Enlace al componente original

Descripción

El Tooltip es un elemento emergente que proporciona información adicional o contextual sobre un componente cuando se interactúa con él.

Uso del componente

El Tooltip se utiliza para brindar información breve y complementaria sin sobrecargar la interfaz, además, su contenido debe ser conciso y relevante.

Se recomienda no depender exclusivamente del Tooltip para transmitir información crítica, ya que algunas personas usuarias pueden no interactuar con él.

Diseño y Formato del componente

Se muestra como una caja pequeña que contiene texto explicativo. Su diseño debe ser discreto pero claro, y suele estar ubicado cerca del elemento que describe. El texto debe ser fácil de leer, sin ser demasiado grande ni pequeño.

Criterios de accesibilidad

Criterios Transversales

  1. Contraste: El texto dentro del Tooltip debe cumplir con los estándares WCAG 2.1 AA o AAA para garantizar su legibilidad en distintas condiciones de visualización. Se recomienda un mínimo de 4.5:1 de contraste entre el texto y el fondo.

  2. Navegación por teclado: El Tooltip debe ser accesible mediante teclado, permitiendo activarlo con Tab (focus) y cerrarlo al mover el foco a otro elemento. Debe ser posible activarlo mediante hover (ratón) y desactivarlo al retirar el cursor. Si el Tooltip contiene información esencial, se debe proporcionar una alternativa persistente para quienes no pueden usar el hover.

  3. Atributos ARIA: Usar aria-describedby para asociar el Tooltip con su elemento activador. Si el Tooltip proporciona información a un icono sin texto visible, se recomienda aria-label para describirlo adecuadamente.

  4. Compatibilidad con lectores de pantalla: Los lectores de pantalla deben anunciar el contenido del Tooltip cuando el foco esté en el elemento activador. Si el Tooltip proporciona datos críticos, su contenido debe poder ser leído sin depender de interacción con el mouse.

Criterios Específicos

  1. Etiqueta Visible: El contenido del Tooltip debe ser claro y descriptivo, permitiendo que se comprenda el mensaje sin necesidad de contexto adicional.

  2. Foco Visible: Si el Tooltip está asociado a un elemento interactivo (ejemplo: un botón), debe ser accesible mediante teclado y su activador debe mostrar un foco visible (como un contorno o cambio de color) para indicar su estado.
