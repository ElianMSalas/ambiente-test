Radio Button

Enlace al componente original

Descripción

Un radio button es un elemento de selección que permite elegir una única opción dentro de un grupo de opciones mutuamente excluyentes.

Uso del componente

Se emplean en formularios, configuraciones y encuestas para garantizar elecciones exclusivas.

Diseño y Formato del componente

Los radio buttons se presentan como círculos pequeños acompañados de una etiqueta descriptiva. Se componen de dos elementos clave: el botón circular de selección y el texto de la etiqueta, que indica la opción correspondiente.

Criterios de accesibilidad

Criterios Transversales

  1. Contraste: El texto de la etiqueta y el estado visual del radio button deben cumplir con los estándares de contraste de la WCAG (AA o AAA) para garantizar la legibilidad en diferentes condiciones de visualización. Este es un aspecto importante para usuarios con dificultades visuales.

  2. Compatibilidad con lectores de pantalla: Los lectores de pantalla deben anunciar el grupo de botones de radio y el estado seleccionado utilizando el atributo aria-checked, lo que garantiza que las personas con discapacidad visual puedan interactuar de manera efectiva con los radio buttons.

  3. Navegación por teclado: Los botones de radio deben ser seleccionables con las teclas de flecha y la tecla Enter. Esto facilita su uso en formularios donde el usuario puede navegar con el teclado sin necesidad de un ratón.

Criterios Específicos

  1. Etiqueta visible: Asegura que la etiqueta asociada al radio button sea descriptiva y clara, permitiendo a las personas usuarias comprender la opción sin necesidad de contexto adicional.

  2. Atributos ARIA: Es importante asociar cada radio button con un grupo mediante los atributos aria-labelledby o aria-describedby, de modo que los usuarios con tecnologías de asistencia puedan identificar correctamente qué grupo de opciones se está presenciando y cuál está seleccionada.

  3. Lectores de pantalla: Utiliza los atributos aria-label o aria-labelledby para proporcionar información adicional cuando sea necesario, asegurando que las personas usuarias con lectores de pantalla puedan interpretar correctamente las opciones disponibles. Además, asociar la etiqueta correctamente al radio button con el atributo for en el label o mediante aria-labelledby.
