Select

Enlace al componente original

Descripción

El Select se presenta como un menú despegable de opciones predefinidas, que permite a la persona usuaria seleccionar una o varias opciones según sea el caso.

Uso del componente

El Select se emplea cuando es necesario ofrecer una lista de opciones acotada. Su uso es ideal cuando las opciones disponibles no pueden representarse de manera eficiente con botones de radio o casillas de verificación.

Casos de uso recomendados:

Formularios donde la persona usuaria debe seleccionar una categoría o estado.

Configuraciones de perfil o preferencias.

Filtrado de información en tablas o listados.

Buenas prácticas:

Evitar listas de opciones demasiado extensas; si hay muchas opciones, considerar un Autocomplete.

Proporcionar una opción predeterminada clara, como "Selecciona una opción".

Mantener consistencia en el orden y la agrupación de las opciones para facilitar la navegación.

Diseño y Formato del componente

Un select se compone de los siguientes elementos:

  • Campo desplegable: Contiene al placeholder y está destinado a mostrar la opción seleccionada.

  • Placeholder: Texto dentro del campo que describe o sugiere un termino o una acción, por ej: "Selecciona".

  • Menú de opciones: Se despliega al interactuar con el campo.

  • Icono de despliegue: Suele ser de forma triangular, indica que el campo se puede desplegar.

  • Helper: Texto utilizado para la muestra de mensajes de error o ayuda.

Variantes del componente:

  • Select Estándar: Permite elegir una sola opción.

  • Select Múltiple: Permite seleccionar varias opciones con multiple="true".

  • Select con Grupos: Organiza opciones en categorías mediante mat-optgroup.

Legibilidad del contenido del componente:

  • El ancho del select debe permitir la lectura de valores.

  • Dependiendo del contexto de uso, se debe tener un ancho mínimo.

Criterios de accesibilidad

Criterios Transversales

  1. Contraste: Asegurar que el texto dentro del select y sus opciones tenga suficiente contraste.

  2. Navegación por teclado:

    • Abrir el Select con Tab o Enter.

    • Navegar entre opciones con Flechas arriba/abajo.

    • Seleccionar una opción con Enter o Space.

    • Cerrar el menú con Esc, sin perder la selección realizada.

    • En el caso de un select múltiple, debe permitir seleccionar/deseleccionar opciones con Space, usar flechas arriba/abajo para navegar entre opciones y shift + flechas para seleccionar un rango.

  3. Atributos ARIA:

    • Si la etiqueta no está visible, se debe utilizar aria-labelledby para asociar el campo con una descripción adecuada.

    • Usar aria-expanded="true/false" para indicar si el menú está abierto o cerrado.

    • Usar aria-activedescendant para resaltar la opción activa.

    • Si se usa un Select múltiple, aplicar aria-multiselectable="true".

    • Cada opción seleccionable debe incluir aria-selected="true" cuando corresponda.

  4. Compatibilidad con lectores de pantalla:

    • El lector de pantalla debe anunciar el Select, la opción seleccionada y las opciones disponibles.

    • Notificar cuando se abre el menú (aria-expanded).

    • Las opciones deben ser leídas cuando se navega a través de ellas.

Criterios Específicos

  1. Etiquetado HTML semántico:

    • No usar aria-labelledby si ya hay un mat-label.

    • Select múltiple (multiple="true") y aria-multiselectable: Cuando se permite la selección múltiple, se debe indicar mediante aria-multiselectable="true" para que los lectores de pantalla lo interpreten correctamente.

    • Estados de selección (aria-selected): Cada mat-option debe incluir aria-selected="true" cuando está seleccionada para que los lectores de pantalla informen correctamente el estado de la opción.

    • Manejo de errores y validaciones: Si el Select requiere validaciones, se debe proporcionar retroalimentación accesible con aria-describedby o aria-live="assertive" para que los lectores de pantalla lo anuncien correctamente.

  2. Uso del color:

    • No se debe depender únicamente del color para indicar estados (activo, deshabilitado, seleccionado, error).

    • Se recomienda combinar color con otros indicadores visuales, como subrayado, íconos o cambios en el grosor del texto.

    • Para estados de error, además del color rojo, se debe proporcionar un mensaje textual de error accesible mediante aria-describedby.
