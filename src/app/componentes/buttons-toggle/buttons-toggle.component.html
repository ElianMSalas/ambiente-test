Descripción

El Button Toggle es un componente que permite alternar entre distintos estados dentro de un conjunto de opciones independientes. Funciona como un grupo de botones en los que puede seleccionar uno o varios dependiendo de la configuración. Desde la perspectiva de accesibilidad, los Button Toggle deben ser correctamente etiquetados y permitir la navegación con teclado para garantizar una experiencia inclusiva.

Uso del componente

El componente Button Toggle se usa cuando se necesita ofrecer opciones de selección visualmente claras, por ejemplo en:

  • Filtros y configuraciones de vista.

  • Opciones de selección de formatos o estilos (ej. negrita, cursiva en un editor de texto).

  • Activación o desactivación de opciones dentro de un formulario.

Buenas prácticas:

  • Utilizar etiquetas claras y descriptivas.

  • Evitar el uso exclusivo de iconos sin texto adicional.

  • Asegurar que el estado seleccionado sea perceptible mediante diseño y atributos ARIA.

Diseño y Formato del componente

El Button Toggle está compuesto por los siguientes elementos:

  • Grupo de botones : Contenedor que agrupa los toggles cuando se requiere una selección exclusiva o múltiple.

  • Botón individual: Representa cada opción seleccionable dentro del grupo.

  • Etiqueta o icono: Cada botón puede incluir texto, un icono o ambos para indicar su función.

  • Estados visuales: Indican si una opción está seleccionada (activo), no seleccionada (inactivo) o si el grupo permite múltiples selecciones.

Criterios de accesibilidad

Criterios Transversales

  1. Contraste: Diferenciar visualmente los estados de selección usando color y/o iconografía.

  2. Navegación por teclado: Navegable mediante Tab, estado debe alternarse con las teclas Enter y Space.

  3. Atributos ARIA: Si los botones tienen solo iconos, incluir aria-label para describir su función.Para toggles múltiples, usar aria-pressed="true/false" en cada botón para indicar el estado de selección.

  4. Compatibilidad con lectores de pantalla: Los lectores de pantalla deben anunciar el estado del toggle.

Criterios Específicos

  1. Etiquetado HTML Semántico: Utilizar button con atributos adecuados. Utilizar role="group" si los botones deben tratarse como un conjunto.

  2. Foco visible: Cuando un botón recibe foco, debe mostrarse un contorno claro.
